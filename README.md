# Klipsch India — CRO Concept Demo

A static, shareable prototype of an **enhanced** Klipsch India (Cinebels) storefront,
built to demonstrate conversion-rate-optimization (CRO) ideas for the team. It uses
**real product data** captured from the live store's catalog API and rebuilds the
experience with the enhancements baked in natively (not injected at runtime).

> **This is a concept demo, not the live store.** Checkout is simulated — no real
> orders are placed and no payments are taken. The live store is operated by
> Cinebels (Cinerama Pvt. Ltd.), the authorised Klipsch distributor in India.

## What it demonstrates

The enhancements we discussed, built in as first-class features:

1. **Buying Guide on the PDP** (the headline feature)
   - A compact **"Quick Verdict + Best for"** strip directly below the price.
   - A detailed **Buying Guide** card below the buy box: *Specs at a glance*,
     *What the features mean* (plain-language jargon buster), and *Good to know*
     (warranty / returns / payment / delivery).
   - All generated by a **deterministic rules engine** from each product's own data —
     no external data, no runtime LLM.
2. **Compare** — select up to 4 products; a side-by-side modal compares price, EMI,
   best-for and specs (with graceful blanks for missing fields).
3. **Offers on the PDP + one-tap apply** — eligible offers shown on the product page,
   applied without coupon-hunting, and reflected in the cart.
4. **EMI surfaced everywhere** — "EMI from ₹X/mo" on cards and PDPs.
5. **Simplified login for un-recognized users** — phone+OTP (India default), Amazon
   sign-in, and true guest / COD paths. Auth is deferred until order placement.
6. **Product finder** ("Find your Klipsch") on the homepage — shop by use case.
7. **Faceted discovery** on the shop page — filter by use case, category, price; sort.
8. **Trust & risk-reversal** — authorised-distributor / warranty messaging, reframed
   returns, delivery pincode check.
9. **Responsive** desktop + mobile, with a **sticky product gallery** on desktop.

## Run locally

No build step or dependencies for the site itself.

```bash
# from this folder
python -m http.server 8099
# then open http://localhost:8099/index.html
```

Or just open `index.html` in a browser (works from `file://` too).

## Deploy on GitHub Pages

1. Push this folder to a GitHub repo.
2. Repo → **Settings → Pages** → Source: `Deploy from a branch` → branch `main`,
   folder `/ (root)` (or `/docs` if you move it there).
3. Your team gets a live link like `https://<user>.github.io/<repo>/`.

## Regenerating the data

Product data was captured from the live SmartBiz storefront API
(`api.smartbiz.in` / store `62469`, `featuredProductsList` payload — real, public
product info only). To rebuild `assets/data.js` after editing `build/raw_products.json`:

```bash
python ../scripts/build_klipsch_data.py
```

The generator parses each product's description into structured specs + feature
bullets, and derives the form factor, "best for" use-case tags, jargon explanations
and the quick verdict — this is the buying-guide rules engine.

## Notes & honest caveats

- **Images** are hot-linked from Amazon's CDN (same source the live store uses). If an
  image fails to load it's a CDN/hotlink issue, not a code bug.
- **Data scope:** the demo ships with 10 real bestseller products across all 5
  categories. The live catalog is larger; the generator scales to whatever is in
  `raw_products.json`.
- **Offers, ratings, EMI amounts** are illustrative placeholders for the demo (clearly
  framed as such), since those aren't all exposed in the public catalog payload.
- This is a **front-end concept** to align the team on UX direction. Production should
  implement these patterns in the actual storefront stack, server-rendered.
